# The Writer — v1.0 Closeout

**Date:** 2026-07-05 (updated same day — repo renamed, package name synced)
**Status:** Production
**Formerly:** Coffee & Curiosity Writing Engine
**Repo:** `github.com/TheUnquietRealms/thewriter` (renamed from `coffee-curiosity-engine`)

---

## What this is

A single-user, local-first writing studio. Everything lives in the browser's
localStorage — no backend, no accounts, no server cost, no data leaving the
machine except direct calls to whichever AI provider you supply a key for.

This document closes out the v2 build (see `2026-06-03-cce-v2-design.md` and
its implementation plan, both fully checked off) and defines what "done"
means for v1.0 under the new name.

---

## MVP — the loop that has to work

The smallest version of this product that is actually usable for writing an
article start to finish. All of this shipped in v0.1–v0.2 and is confirmed
working in this closeout:

- Create an article, pick a writing mode (Essay, Fiction, Book Chapter, etc.)
- Write in a distraction-light editor, autosaved to localStorage on every
  change — nothing is lost on refresh or browser close
- Organise: title, subtitle, body, status (draft/review/published)
- Get it out: copy Markdown, export `.md`, export `.docx`
- Full backup/restore: export all data to a JSON file, re-import it
  elsewhere (this is the answer to "what if I lose my browser profile" —
  it's manual, but it exists and works)

**Verdict: MVP is solid.** This is the part that can never break, and
nothing in this closeout touches it.

---

## MLP — the layer that makes it worth opening every day

Everything on top of the MVP that turns "a text box that saves" into a tool
with an opinion about your writing. Shipped in v0.2.1–v0.3.0:

- **Codex** — voice rules, banned AI habits, recurring themes, source notes,
  publication checklist. Per-article overrides for voice/banned habits.
- **Editorial review** — deterministic score ring (not AI-judged — rule-based,
  so it's fast, free, and repeatable) with mode-specific categories.
- **Grammar check** — LanguageTool integration, one-click apply per match.
- **Focus tools** — Pomodoro timer, focus mode, markdown preview toggle.
- **Organisation** — tags, project grouping (this is what makes book chapters
  work: one project, many chapters-as-articles), full-text search.
- **AI assist** — four providers (Gemini, OpenAI, OpenRouter, Anthropic),
  user-supplied key, never transmitted anywhere but the chosen provider.
  Continue / suggest outline / brainstorm / research, all constructed from
  the Codex so suggestions respect your stated voice rules and banned habits.

**Verdict: MLP is solid.** This is a genuinely well-scoped feature set for a
single-user tool — nothing here is a stub or a half-feature.

## What this closeout actually changed

1. **Rebrand**: "Coffee & Curiosity Writing Engine" → **The Writer**, across
   the page title, navigator header, onboarding screen, focus-timer
   notification, and the OpenRouter request headers. Deliberately *not*
   changed: localStorage key names (`cce_*`) — renaming those would have
   orphaned every article already saved in your browser.
2. **GitHub repo renamed** to `thewriter` (from `coffee-curiosity-engine`).
   The npm package `name` field in `package.json` has been synced to match
   (`"thewriter"`). GitHub's redirect keeps the old repo URL working, but
   new links should use the `thewriter` path. The `pages.dev` subdomain is
   a separate Cloudflare Pages resource and is still unchanged (see Epic 5).
2. **Type-checking was silently broken.** `tsc --noEmit` failed with ~40
   errors because `@types/react` and `@types/react-dom` were never installed
   — Vite's build doesn't need them to compile, so this went unnoticed.
   Added both, plus a `npm run typecheck` script. Confirmed clean.
3. **Version bumped** to 1.0.0 to mark this as the closed-out production
   release rather than an ongoing v0.x build.

## Epics — real remaining work, not yet done

These are the things worth naming honestly rather than pretending the
project has zero further work. None of them block using the tool today.

### Epic 1 — Bundle size
Single 653 KB JS chunk (197 KB gzipped). `docx`, `marked`, and four AI
provider clients all ship to every visitor regardless of use. Fix is
routine: dynamic `import()` for the export and AI modules, or
`manualChunks` in `vite.config.ts`. Low risk, an afternoon of work.

### Epic 2 — Zero automated tests
There is no test suite — nothing in `package.json`, no `*.test.ts` files.
For a single-user tool this hasn't caused visible problems, but it means
every future change is verified by hand. Worth at minimum a handful of
unit tests around `storage.ts` (the localStorage read/write/migration
logic is exactly the kind of code that breaks silently) and `review.ts`
(the scoring logic).

### Epic 3 — Single-device data
By design, articles live in one browser's localStorage. The export/import
backup feature is the current answer, but it's manual. If daily use across
more than one device becomes real, the actual fix is a thin sync layer
(even just "push backup JSON to a private GitHub gist on save" would beat
nothing), not a full backend.

### Epic 4 — Publish integration with Said Plainly
You now have two systems: this writer, and the live opinion site
(zencloudau.github.io/CoffeeandCuriosity, rebranded to *Said Plainly*).
Right now getting an article from one to the other is copy-paste. A real
epic here: an "Export → Said Plainly HTML" button that wraps the article
body in the site's existing template (chapter number, deck, date, pager)
and downloads a ready-to-commit `.html` file, or further out, pushes it to
the GitHub repo directly the way this session did manually. This is the
highest-leverage epic if the goal is writing articles *for publication*,
not just writing.

### Epic 5 — Domain rename (partially closed)
GitHub repo and npm package name are now both `thewriter` — done. The one
piece still outstanding is `coffee-curiosity-engine.pages.dev`, which still
carries the old name. Renaming the Cloudflare Pages project (or attaching a
custom domain) is straightforward but is a Cloudflare dashboard action, not
a code change — flagging it here so it doesn't get lost, not doing it
silently.

---

## Bottom line

MVP: done. MLP: done. This closeout fixed one real bug (broken
type-checking) and did the rebrand cleanly without touching stored data.
Epics 1–5 are the honest list of what's left, ordered roughly by how much
they'd actually change your day-to-day use of the tool — Epic 4 (publish
integration) is the one worth doing next if the goal is feeding this
straight into your writing.
